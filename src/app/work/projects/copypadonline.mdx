## Overview

**CopyPad** is a frictionless, URL-based text-sharing platform designed for instant document creation and collaboration — without authentication barriers.

Users can create a shared document simply by navigating to a custom path (e.g., `/notes`, `/project/docs`). The platform auto-saves content in real time, supports collaborative editing via WebSockets, and works seamlessly across devices.

The philosophy is simple: **zero onboarding, zero complexity, instant productivity.**

---

## Core Objectives

- Enable instant document creation without login
- Provide real-time collaborative editing
- Maintain lightweight, fast-loading architecture
- Support multiple storage strategies (File-based + MongoDB)
- Ensure strong baseline security against abuse

---

## Key Features

### 🔗 URL-Based Document Creation

- Dynamic document generation via custom routes
- Supports hierarchical paths (`/project/frontend/ideas`)
- Case-sensitive routing for granular control

### 💾 Intelligent Auto-Save

- 800ms debounce auto-save mechanism
- Version-controlled updates
- Backend persistence through file system or MongoDB

### 🔄 Real-Time Synchronization

- Powered by WebSockets using Socket.io
- Room-based document sessions
- Instant content updates across connected users

### 📊 Live Editor Stats

- Real-time word and character counter
- Version tracking metadata
- Timestamp-based modification records

### 🌙 Adaptive Theming

- System-based dark/light mode detection
- Manual toggle with persistent localStorage state
- Clean, distraction-free UI

### 📥 Export & Sharing

- One-click URL copy
- Download documents as `.txt` files
- Simple share-by-link workflow

## Architecture & System Design

CopyPad follows a modular full-stack structure:

### Backend

- **Node.js** runtime
- **Express.js** server
- RESTful API endpoints (`/api/doc/:path`)
- Socket.io WebSocket layer for real-time sync

### Storage Options

| Storage Type | Description | Configuration |
|--------------|-------------|--------------|
| File Storage (Default) | JSON files stored locally | No setup required |
| MongoDB | Mongoose-based schema storage | Enable via environment variables |

File-based storage ensures simplicity, while MongoDB enables scalable deployments.

---

## API Layer

### Get Document
`GET /api/doc/:path`

Returns document content, version, and last modified timestamp.

### Save Document
`POST /api/doc/:path`

Updates content with version increment.

### Document Info
`GET /api/doc/:path/info`

Returns metadata including size, creation date, and modification history.

---

## Real-Time Engine

Socket.io enables collaborative editing:

- Users join document-specific rooms
- Broadcast updates on content change
- Maintain synchronization across sessions

This ensures minimal latency and consistent document state.

---

## Security Implementation

- Input sanitization to prevent XSS attacks
- 1MB document size cap to prevent abuse
- URL-based access model (privacy by obscurity)
- Environment-based configuration control

Security is lightweight but practical for a minimal sharing tool.
