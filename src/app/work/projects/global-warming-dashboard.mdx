## Overview

**GreenPulse — Climate Data Explorer** is a responsive web dashboard built to visualize key climate indicators using real-world environmental datasets.

The project focuses on transforming complex climate data into clear, interactive visuals that raise awareness about global warming and environmental change. Designed with a clean, accessible interface, GreenPulse balances educational storytelling with modern frontend engineering practices.

---

## Core Objectives

- Present climate datasets in an intuitive, visual format  
- Increase accessibility and awareness around global warming trends  
- Build a performant dashboard optimized for real-time data updates  
- Maintain responsive usability across devices  
- Implement modern UI practices with smooth interaction flows  

---

## Key Features

### 📊 Interactive Climate Data Visualization

- Global land and ocean temperature trends
- Atmospheric greenhouse gases:
  - CO₂ (Carbon Dioxide)
  - CH₄ (Methane)
  - N₂O (Nitrous Oxide)
- Polar sea ice extent tracking

Charts are interactive and optimized for clarity and exploration.

### 🌦️ Live Weather Integration

- Real-time weather widget
- 7-day forecast powered by Weatherbit API
- Collapsible interface for smaller screens

Note: Weatherbit API free-tier request limits may temporarily restrict forecast visibility without affecting core dashboard functionality.

### 📱 Responsive Experience

- Mobile-first design approach
- Automatic chart switching to bar charts on smaller devices
- Clean navigation using Hero UI components

### ⚡ Performance & UX Enhancements

- Smart caching to minimize redundant API requests
- Animated loading states and skeleton placeholders
- Smooth motion transitions powered by Framer Motion
- Export charts as PNG or PDF for reporting and sharing

---

## Architecture & Development Approach

The project follows a structured, modular architecture:

### Project Structure Highlights

- API clients isolated within dedicated modules
- Functional React components using hooks
- Custom hooks for responsive behavior and caching
- Layout wrappers for consistent UI structure
- Component-level organization for scalability

Custom hooks include:

- `useCachedData` — Reduces API requests through intelligent caching
- `useIsMobile` — Enables adaptive responsive logic

- Clean, minimal dashboard layout
